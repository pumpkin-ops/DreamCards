# Project Justification

DreamCards has open-source value because it combines several problems that are usually developed in isolation: multimodal AI behavior, user-generated visual content, hidden-information multiplayer rules, creator provenance, and content moderation. The project is not only a browser adaptation of an image-association game. Its core research question is how an image can move through a durable community lifecycle: creation, discovery, collection, curation, play, reinterpretation, and replay. Publishing the implementation and design decisions makes that lifecycle inspectable and reusable by developers working on social games, creative communities, AI-assisted play, and UGC systems.

The maintenance burden is substantial enough to benefit from an OSS maintainer support program. AI providers are probabilistic and operationally unreliable: outputs must be parsed, validated, timed out, constrained by game rules, and replaced by deterministic local behavior when necessary. UGC introduces provenance, copyright, abuse-reporting, policy-versioning, and moderation-queue requirements. Multiplayer introduces authoritative state, reconnect behavior, duplicate-action prevention, hidden-information boundaries, and race conditions. These systems interact: a delayed AI vote must not reveal another player's choice, a rejected image must not enter a shared deck, and a reconnecting client must receive only the information allowed in the current phase.

Codex could improve maintainer capacity in practical, reviewable ways. It can help triage bug reports into reproducible state-machine cases, draft regression tests from confirmed failures, identify missing validation at API boundaries, and review pull requests against repository-specific rules in `AGENTS.md`. For moderation work, Codex can assist with policy-to-test translation, audit-log schema review, and consistency checks across upload, report, quarantine, and appeal paths without making final moderation decisions. For routine maintenance, it can update documentation when interfaces change, detect duplicated provider logic, propose small refactors, and summarize risk areas for human reviewers.

Support would therefore accelerate work that is difficult for a small maintainer team but broadly useful to the open-source ecosystem: dependable AI fallbacks, transparent UGC safety layers, testable information-permission rules, and multiplayer systems designed around partial asynchronous completion rather than blocking global state.
