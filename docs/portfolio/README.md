# DreamCards 作品集项目说明

> 项目类型：AI 联想叙事桌游 / UGC 图片社区 Demo  
> 目标岗位：游戏系统策划、玩法策划  
> 项目阶段：可交互原型，核心单人流程可完整运行  
> 文档版本：2026-06-08

## 一句话介绍

DreamCards 是一款围绕图片联想、故事创作与赛后讨论设计的四人桌游。玩家既是对局参与者，也是梦境图片的创作者、发现者与策展人；单人模式中，三名视觉模型 AI 会直接观察图片并完成说书、出牌和投票。

## 我在项目中的职责

本项目由我负责产品方向与系统设计，并持续通过可交互 Demo 验证规则。

- 定义“图片联想 + UGC 创作 + 收藏发现”的产品定位。
- 拆解并实现说书、跟牌、匿名展示、投票、计分和轮换的完整回合。
- 设计卡牌身份、创作者署名、收藏、图鉴和梦境集系统。
- 设计游戏内信息权限，避免名称、标签和创作者信息干扰联想。
- 设计视觉模型 AI 的说书、跟牌、投票和异常降级规则。
- 通过实玩发现同步阻塞、信息泄露、重复卡牌等问题，并推动规则和交互闭环。
- 约束 UI 方向，使牌桌、图片和玩家行为优先于后台式信息展示。

## 核心设计命题

### 1. 如何把一次性桌游变成长期系统？

传统联想桌游的主要价值集中在单局体验。DreamCards 在对局外加入：

`创作图片 → 被其他玩家发现 → 收藏作品 → 策展梦境集 → 带入对局 → 产生新联想`

由此形成内容生产、内容消费和社交传播闭环。

### 2. 如何让 AI 像玩家，而不是随机机器人？

三名 AI 使用不同视觉模型，直接读取图片：

- 说书：生成中等模糊度提示，目标是让部分玩家猜中。
- 跟牌：选择与提示有关、但不过于明显的图片。
- 投票：排除自己的牌后判断说书人的图片。
- 异常：超时、限流或格式错误时，单个 AI 独立降级，不阻塞整局。

### 3. 如何处理实时桌游中的异步行为？

玩家的操作不能被其他玩家的网络或思考速度阻塞。因此对局采用逐玩家状态：

- 玩家出牌后立即更新自己的座位与手牌。
- AI 在后台独立行动，谁完成，谁先在桌前放下牌背。
- 投票同样逐人更新，但揭晓前不公开投票目标。
- 所有必要玩家完成后自动结算。

这套设计同时满足即时反馈、公平性和信息保密。

## 原型阶段量化结果

以下数据来自 2026-06-08 的单轮功能基准测试，不代表线上大样本表现：

- 玩家说书提交从约 20 秒同步等待，优化为约 3ms 立即返回。
- 玩家在 AI 说书轮跟牌约 1ms 返回。
- AI 行为由批量刷新改为逐人可见。
- 单模型设置 9 秒硬超时，避免无限阻塞。
- 一次完整 AI 回合在 8943ms 内自动揭晓。
- 四名玩家共 40 张卡牌实现 `40 cardId / 40 images` 唯一分配。

## 三分钟演示建议

1. 用 20 秒说明“图片、联想、故事、围桌体验”的产品定位。
2. 展示无标题卡牌与固定围桌布局。
3. 提交一张说书牌，指出自己的状态立即更新，AI 牌背逐个出现。
4. 展示匿名投票、禁止自投和逐玩家投票状态。
5. 进入结算，说明计分为什么服务于提示模糊度和干扰牌价值。
6. 最后展示图鉴、永久署名和梦境集，说明对局如何连接长期内容循环。
